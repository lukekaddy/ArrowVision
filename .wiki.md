# Project Summary
**BullsEye Labs** is a mobile-first web application designed to revolutionize archery tournament management through advanced digital scorekeeping. It provides a comprehensive suite of tools for tournament organizers, archers, and scorekeepers, including live scoring, real-time leaderboards, and instant results. The application supports diverse tournament formats and features advanced scoring capabilities, including a "Replay Camera" for video analysis of arrow impacts. Recent updates have focused on enhancing user experience, optimizing video uploads, and improving navigation.

# Project Module Description
- **Authentication**: Role-based email/password authentication using bcrypt with local caching.
- **Tournament Management**: Tools for creating and managing tournaments with course naming and target specifications.
- **Live Scoring**: Real-time score updates and input.
- **Leaderboard**: Dynamic leaderboard displaying current rankings with course filtering.
- **Smart Score**: AI-assisted scoring with video replay functionality, now includes an authorization token for score submissions.
- **Results Export**: Export tournament results in JSON format.
- **Archer Registration**: Registration forms for user details, division selection, and group assignments, with enhanced mulligan options.
- **Replay Camera**: Captures and uploads video replays triggered by visual motion detection, now with improved stabilization and sensitivity settings.
- **Create/Edit Scorecard**: Users can create and edit scorecards linked to tournaments.
- **Improved Navigation**: Enhanced navigation for better access to functionalities, including direct links to scorecards.
- **Course Map Upload**: Users can upload course map images during tournament creation.
- **Archer Experience**: Pages for archers to view registered tournaments and scorecard history.
- **Profile Management**: Profile page for users to manage accounts and switch roles.
- **Public Tournaments Endpoint**: Added an alias route for fetching public tournaments.
- **Debugging Tools**: New endpoint for diagnosing missing records in video uploads.
- **Video Playback Improvements**: Enhanced error logging and preflight checks for video playback.

# Technology Stack
- **Frontend**: React, TypeScript, Vite, Tailwind CSS, shadcn/ui
- **Backend**: Atoms Cloud (FastAPI, PostgreSQL, Authentication)
- **SDK**: @metagptx/web-sdk for authentication and API calls

# Usage
1. **Install Dependencies**:
   - For the backend: Run `pip install -r requirements.txt`
   - For the frontend: Run `pnpm install`
   
2. **Build the Project**:
   - For the backend, ensure the database is set up and migrations are applied.
   - For the frontend, run `pnpm run build` to compile the application.

3. **Run the Application**:
   - Start the backend server.
   - Start the frontend server to access the web application.
