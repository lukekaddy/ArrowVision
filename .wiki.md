# Project Summary
**BullsEye Labs** is a mobile-first web application designed to streamline archery tournament management through digital scorekeeping. It serves tournament organizers, archers, and scorekeepers, providing features like live scoring, real-time leaderboards, and instant results. The platform supports various tournament formats and advanced scoring options, enhancing user experience with tools such as the "Replay Camera," which allows users to capture and upload video replays of arrow impacts. Recent enhancements include a Profile page for users to manage their accounts and switch roles between Tournament Archer and Tournament Admin.

# Project Module Description
- **Authentication**: User authentication via Atoms Cloud with role-based access.
- **Tournament Management**: Tools for creating, editing, and managing tournaments, including course naming and target specifications.
- **Live Scoring**: Real-time score updates and input.
- **Leaderboard**: Dynamic leaderboard with current rankings and course filtering.
- **Smart Score**: AI-assisted scoring with video replay functionality.
- **Results Export**: Export tournament results in JSON format.
- **Archer Registration**: Registration form for user details, division selection, and group assignments.
- **Replay Camera**: Captures and uploads video replays triggered by sound detection.
- **Create/Edit Scorecard**: Allows users to create and edit scorecards linked to tournaments.
- **Improved Navigation**: Restructured main navigation to include Home, Create Tournament, Create/Edit Scorecard, Live Leaderboard, Results, Replay Camera, and Scorecard.
- **Course Map Upload**: Users can upload course map images when creating tournaments.
- **Archer Experience**: New pages for archers to view their registered tournaments and scorecard history.
- **Profile Management**: New Profile page for users to view their account details and manage role switching.

# Technology Stack
- **Frontend**: React, TypeScript, Vite, Tailwind CSS, shadcn/ui
- **Backend**: Atoms Cloud (FastAPI, PostgreSQL, Authentication)
- **SDK**: @metagptx/web-sdk for authentication and API calls

# Usage
1. **Install Dependencies**:
   - For the backend: Run `pip install -r requirements.txt`
   - For the frontend: Run `pnpm install`
   
2. **Build the Project**:
   - For the backend, ensure the database is set up and migrations are applied.
   - For the frontend, run `pnpm run build` to compile the application.

3. **Run the Application**:
   - Start the backend server.
   - Start the frontend server to access the web application.
