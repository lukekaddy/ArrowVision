# Project Summary
**BullsEye Labs** is a mobile-first web application that transforms archery tournament management by digitizing scorekeeping. It offers a comprehensive platform for tournament organizers, archers, and scorekeepers, featuring live scoring, real-time leaderboards, and instant results. The application supports various tournament formats and advanced scoring options, including mulligans. Recent updates have introduced enhanced user interface elements, realistic video replays of arrow impacts, and a role-based access system for streamlined user login. The latest enhancements include the ability to edit and delete scorecards, alongside simplified scorecard selection and course map upload functionalities.

# Project Module Description
- **Authentication**: User authentication via Atoms Cloud with role-based access.
- **Tournament Management**: Tools for creating, editing, and managing tournaments, including course naming and target specifications.
- **Live Scoring**: Real-time score updates and input.
- **Leaderboard**: Dynamic leaderboard with current rankings and course filtering.
- **Smart Score**: AI-assisted scoring with video replay functionality.
- **Results Export**: Export tournament results in JSON format.
- **Archer Registration**: Registration form for user details and division selection.
- **Replay Video Upload**: Functionality for admins to upload video replays tied to specific users and targets.
- **Create/Edit Scorecard**: Users can create and edit scorecards linked to tournaments, now featuring a photo upload for thumbnails instead of a URL input.
- **Improved Navigation**: Restructured main navigation to include Home, Create Tournament, Create/Edit Scorecard, Live Leaderboard, and Results.
- **Course Map Upload**: Functionality allowing users to upload course map images when creating tournaments.

# Technology Stack
- **Frontend**: React, TypeScript, Vite, Tailwind CSS, shadcn/ui
- **Backend**: Atoms Cloud (FastAPI, PostgreSQL, Authentication)
- **SDK**: @metagptx/web-sdk for authentication and API calls

# Usage
1. **Install Dependencies**:
   - For the backend: Run `pip install -r requirements.txt`
   - For the frontend: Run `pnpm install`
   
2. **Build the Project**:
   - For the backend, ensure the database is set up and migrations are applied.
   - For the frontend, run `pnpm run build` to compile the application.

3. **Run the Application**:
   - Start the backend server.
   - Start the frontend server to access the web application.
